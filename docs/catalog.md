# Каталог семейных подписок и тарифов

Источник данных для первого движка SubsMarket:
[family-services.json](../data/family-services.json).

## Правила каталога

- В каталоге находятся только записи для Family Engine:
  `family_type=subscription` и `family_type=tariff`.
- Разовые доступы, аккаунты и гигабайты хранятся отдельно в будущем
  Marketplace Engine.
- Семью можно создать только для активной записи каталога.
- Пользователь не может создать произвольный сервис текстом.
- Лимит участников задается записью каталога и не превышает 8.
- Цена семьи задается владельцем, но не превращается в цену каталога.
- Каталог не должен автоматически наследовать цены из старого проекта.

## Состояния записи

- `pending_verification` - сервис добавлен из исследования рынка, но его
  условия еще не подтверждены.
- `active` - сервис проверен и доступен для создания семей.
- `disabled` - создание новых семей временно запрещено.
- `archived` - сервис больше не предлагается, старые семьи остаются в истории.

## Проверка перед активацией

Для каждого сервиса нужно заполнить:

- официальное или допустимое максимальное число участников;
- доступные платежные периоды;
- регионы использования;
- ограничения семейного плана;
- отображаемое название и логотип;
- признак доступности в Казахстане;
- дату последней проверки.

До заполнения этих данных запись остается `pending_verification` и не
показывается пользователям как доступная для создания семьи.

## Второй движок

Витрина разовых доступов не входит в этот каталог и не входит в первый этап
MVP. Аккаунты и гигабайты позже используют общий `Marketplace Engine` с
раздельными каталогами и типами деталей объявления. Объявление живет до своей
даты окончания, первоначально бесплатно публикуется на 30 дней, может
продлеваться и не закрывается после заявки покупателя.
